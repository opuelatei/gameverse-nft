# Contributing to GameVerse

We love your input! We want to make contributing to GameVerse as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. If you've changed APIs, update the documentation
4. Ensure the test suite passes
5. Make sure your code follows the style guidelines
6. Issue that pull request!

## Smart Contract Modifications

When modifying the smart contract:

1. Always maintain backward compatibility
2. Add proper error handling
3. Include comprehensive tests
4. Document all new functions and changes
5. Follow the rate limiting patterns
6. Maintain security best practices

### Security Considerations

- Never remove or weaken access controls
- Maintain rate limiting on critical functions
- Validate all inputs thoroughly
- Keep asset transfers secure
- Preserve game balance

## Bug Reports

We use GitHub issues to track public bugs. Report a bug by opening a new issue.

**Great Bug Reports** tend to have:

- A quick summary and/or background
- Steps to reproduce
- What you expected would happen
- What actually happens
- Notes (possibly including why you think this might be happening)

## Feature Requests

We love feature requests! Please use the issue tracker to submit feature requests.

When proposing a feature:

1. Explain the problem you're trying to solve
2. Explain how your feature would work
3. Consider the impact on existing functionality
4. Consider security implications
